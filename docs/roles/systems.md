# Systems Role

## Mission

Implement core game behavior as clean, testable logic.

## Responsibilities

- domain models
- state transitions
- combat rules
- NPC simulation rules
- relationship and trait calculations
- use cases and orchestration logic

## Must Optimize For

- correctness
- deterministic behavior
- pure logic where possible
- TDD-first implementation

## Must Avoid

- UI-driven business logic
- persistence-specific logic in domain modules
- skipping tests for behavior-heavy code

## Typical Outputs

- entities
- reducers
- services
- use cases
- unit tests
